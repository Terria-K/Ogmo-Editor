ChangeLog

2.4.0.0

Jan 11, 2018

Changes

  • Removed .jogmo project file format; both XML and JSON projects are now stored as .oep files.
  • Levels will now automatically reopen themselves after editing a project.
  • Added "JSON (2D Array)" and "JSON (1D Array)" as Tilemap export options for JSON projects.
  • JSON serialization improvements:
    • Level files now use the correct data types for storing values instead of serializing everything as a string.
    • Improved readability for small objects (grid layer rectangles, entity nodes, etc).

2.3.0.0

Dec 31, 2018

Changes

  • Added support for JSON Projects:
    • Create a JSON Project from Project > New Project > JSON Project.
    • JSON Projects use *.jogmo project files and *.json level files.
    • XML Projects should work exactly as they did before -- if not, that's a bug!
  • Added "JSON (IDs)" and "JSON (Co-ords)" as Tilemap export options that work similarly to their XML equivalents.
  • Other fixes:
    • Fixed bug where entity selection would persist across projects
    • Changed the default background color of new projects to the classic blue color

2.2.0.0

Dec 28, 2018

Changes

  • First release of the Ogmo Community Version!
  • Added support for macOS and Linux (via Wine)
  • Added zooming to tileset windows (mouse wheel to zoom + RMB/MMB to pan)
  • Features and fixes from the talesofgames ToGmo fork:
    • Much improved rendering times for large maps
    • Improved rendering clarity
    • Improvements to entity scaling
    • Added "Apply Tileset to Layer" button
    • Added Transparent Layers toggle (View > Transparent Layers)
  • Other fixes:
    • Fixed bug where hotkey actions (e.g. copy/cut/paste) would become unresponsive
    • Fixed exception caused by entity value editors losing focus
    • Fixed exception caused by shrinking the tileset window
    • Fixed bug where integer entity value sliders would behave erratically
